ID system used during data collection
(Which will make it A LOT easier to do the bidscoin and fmriprep procedure).
1) In stormDb, create all subjects needed before the workshop starts
Give a fake CPR number that starts with the first 4 digits in the series number that subject will have on stormDb (if the last scan was series number 0069 then the new one will be 0070 and the CPR number should start with 0070).
2) Now, assign each of the CPR-numbers to a study group
This study group will not register their participant with this CPR-number
The SUBID for the psychopy script should be the first 4 digits in the CPR-number (so that it matches with the series number of the MRI images in stormDb).
This way, the logfiles and the MRI images will be named with the same 4 digits for each subject, which make it a lot easier to run the bidscoin and fmriprep procedures as we can match files directly by the first 4 digits in the filename.
Example from workshop 2023:
SUBID is the MRI series number you get when creating a subject
SUBID is used as ID in psychopy script
CPR number is registered when creating the subject on stormDb and is inputted on scanner PC when registering a new patient (under Patient ID).

Steps to run BIDS conversion and fmriprep on FaceWord data collected in CognNeuroSci fMRI workshop
All the scripts and necessary packages/files/dependencies are found in the folder “line_bidscoin_fmriprep”.
Disclaimer!
When reading through this it seems like a lot of work and quite the hazzle.. In reality, once you have done this once it is actually very easy and fast to do (takes about 10 min to do it all and then it’s just waiting for fmriprep to finish )
In general on how to submit to cluster from python
Each of the python scripts have to be submitted to the cluster, so that we don’t run them from the common “workspace”. For this, I have created a script called submit_to_cluster.py”.
All you have to do is define the name of the python script you want to run in the
“script = ‘scriptName’” code line.
You also have the option to change the cluster node to use (depending on how long or heavy the procedure is - for bidscoin you want to use the highmem, for the other processes you can use the short one).
When you have defined which script you want to run, save the changes, and run the script from the terminal:
Navigate to the scripts folder in terminal
(base) python submit_to_cluster.py
Now a job will be submitted to the cluster and compute the python script you have defined.
To check the status of your job
(base) qstat -u <username>
Will show you if it is in queue (qw) or running (r)
If it’s done nothing will pop up when you execute the command
The cluster will output a file called “py-wrapper_xxxxx.qsub” in the scripts folder, which will show you the output of your script that is running. This is useful for e.g., debugging or checking that everything worked as intended.
Workflow
The following steps below runs both bidscoin and fmriprep on FaceWord data from the workshops.
The fmriprep takes quite a while to run, so the easiest thing is to run this workflow by the end of each workshop day. This way we are running about 4 subjects at a time each day and all preproessing will be done the day after the last workshop day.
You can also wait and do it for all subjects in the end, however, then it might take a few days before it is done.
Start: activate virtual environment
If not already activated, activate your bidscoin environment from terminal.
(base) conda activate yourenvname
Copy raw files to scratch
Open script: raw_to_scratch.py
Set appropriate paths (below are the paths used in the workshop 2023)
Raw files path: /projects/Undervisning_CognNeuroSci/raw
Scratch path: /projects/Undervisning_CognNeuroSci/scratch/raw
The raw folder contains all subjects from all years, which we of course don’t want to include. So change the subjects to include in the script. The line that determines which subjects to copy is:
sub_list = [’subID’, ‘subID’, ‘subID’]”
Save changes
Open the submit_to_cluster.py script
Edit the script to be submitted: script = ‘raw_to_scratch.py’
Save changes
Submit to cluster from terminal:
(base) python submit_to_cluster.py
This takes a 1-2 minutes per subject to run.
Run bidsmapper and bidscoiner
Open script bidscoin_converter.py
Set appropriate paths (arguments)
First argument = path to raw folder in scratch: /projects/Undervisning_CognNeuroSci/scratch/raw
Second argument = path to BIDS folder: /projects/Undervisning_CognNeuroSci/scratch/BIDS_line_temp
Third argument = path to bidsmapper .yaml file: /projects/Undervisning_CognNeuroSci/scripts/line_bidscoin_fmriprep/bidsmap_template_line3.yaml
NB! The bidscoin_converter.py will run bidsmapper and bidscoiner on all subjects in folder specified in the first argument - if you want to run it on only a few subjects, you have to move those subjects’ raw files into a separate folder and pass this as the “raw” folder argument to bidsmapper and bidscoiner (the first path specified).
Save changes in script
Open script submit_to_cluster.py
Edit the script to run: script = ‘bidscoiner.py’
Save changes
Submit to cluster from terminal:
(base) python submit_to_cluster.py
This takes about 2 minutes per subject.
Convert event files to BIDS format
This script will copy all logfiles from /aux drive to the BIDS folder, rename each file to match BIDS structure, and store logfile for each of the 6 sessions under the right subject in the BIDS folder.
It will also edit the logfile, so that it contains the first three columns “onset”, “duration” and “trial_type”, which is required by the BIDS format.
For some reason, the bidsmapper does not work, when I try to output it to “BIDS” folder, but when I output to a new folder (here I used “BIDS_line_temp”) it works.
Open script convert_faceWord_log_to_BIDS.py
Edit the subjects you want to include (by inputting right SUBIDs in the file_list1 = “…”).
If you want to include all subjects in a folder you can use the code lines with glob search pattern (.*csv) and list files (currently commented out).
Save changes
Open script submit_to_cluster.py
Edit the script to be run: script = ‘convert_faceWord_log_to_BIDS.py’
Save changes
Submit to cluster from terminal:
(base) python submit_to_cluster.py
This takes a few seconds to run.
Add .json file to eventfiles in BIDS/func folder
The BIDS format requires each of the event.tsv files to have an associated .json files with metadata desscriptions. I have made a standard .json files that match the event files from the FaceWord exp. It is stored in the /scripts folder and named “eventFiles.json”.
The script below will simply copy this json file and add it to each of the event files for each of the subjects in the BIDS folder, with the same name as the json files. This fits with the BIDS standards and necessary for fmriprep to work.
Open the script submit_to_cluster.py
Edit the script to be run: script = ‘add_event_jsonFiles.py’
You can edit the subjects you want to include, but this process is extremely fast so easier to just run it on everyone event if it was already run on some of the subjects.
Save changes
Submit to cluster from terminal
(base) python submit_to_cluster.py
This takes a second to run.
Run fMRIprep
NB! this process is not submitted to the cluster via the submit_to_cluster.py script as the other ones.
Open script run_fmriprep_all.py
Set appropriate path to /scratch/BIDS folder (first path in code)
Set appropriate path to BIDS output folder: /scratch/BIDS/derivatives (second path in code)
NB! You have to create the “derivatives” folder yourself first
Set appropriate path to Freesurfer License (third path in code)
Currently saved in /scripts/line_bidscoin_fmriprep
Set appropriate path to scratch folder (work space) (fourth path in code)
Save changes
This script also contains a function to run smoothing in the preprocessing. I have not used this, and thus not tested exactly how it works. But the code is there if you want to use it.
Open script run_fmriprep_submit.py
Edit which subjects to include: sub_list = [’subID’, ‘subID’, subID’]
(you don’t want to run it on subjects that have already been preprocessed, because this takes a looooonnggg time)
Save changes
Submit to cluster from terminal:
(base) python run_fmriprep_submit.py
This script will submit one job per subject to allow them to run in parallel.
The output will appear in the “derivatives” folder, containing preprocessed anat images, preprocessed func images (and many confounds), and a figures folder with visualization of masks etc. used in preprocessing.
The derivatives folder will also contain a .html file per subject, with a summary of the preprocessing. This cannot be open from vscode but can be opened via jupyter notebook in ucloud when everything has been uploaded to there —> it is super nice for the students to check how the preprocessing went and which steps were taken.
This takes about 4 hours per subject to run.

Alternative solution (what Line used in 2023):
Create account on sciencedata.dk if you don’t already have one
Get your username and password
From terminal on stormDb - tar zip the BIDS folder
$ tar fzc <arkivnavn>.tgz <mappe>/*
Arkivnavn is the name of the resulting zip file
mappe is the folder you want to zip
NB! It is important that you zip with tar zip as showed here. Otherwise it can be problematic to unzip it in ucloud later.
Upload the folder to sciencedata.dk
$ export user=<username>; export kode=<password>
$ curl -k -u ${user}:${kode} -T <folder.tgz> https://sciencedata.dk/files/folder.tgz
From terminal on uCloud
Open a terminal run including the folder where you want to downlaod the data to
Navigate in terminal to that folder
Download the data from sciencedata.dk
$ curl -O -u ${user}:${kode} -G https://sciencedata.dk/files/folder.tgz
Unzip folder
$ tar fxz <arkivnavn>.tgz
Move it via terminal to where you want to keep it
